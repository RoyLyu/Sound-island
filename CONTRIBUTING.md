# Contributing

感谢你帮助声屿变得更适合真实的声音制作工作。

## 提交问题

请说明：

- 操作系统与版本
- 声屿版本
- 可复现步骤
- 预期结果与实际结果
- 如与文件有关，请提供脱敏后的文件名与格式，不要上传无权分享的商业音效

## 开发流程

1. Fork 仓库并从 `main` 创建功能分支。
2. 保持改动聚焦，避免把无关格式化混入功能提交。
3. 运行前端构建与 Rust 测试。
4. 提交 Pull Request，说明改动、原因、验证方式和隐私影响。

```bash
npm install
npm run build
cargo test --manifest-path src-tauri/Cargo.toml
```

## 自动分类规则

分类规则必须尽量保守：无法确定时进入「未分类」优于错误归类。新增规则时请同时添加测试，至少包含：

- 一个应当命中的英文文件名
- 一个应当命中的中文文件名
- 一个不应被误命中的相似文件名

## 代码原则

- 默认本地处理，不新增无必要的网络请求。
- 不静默移动、改名或删除用户音频。
- 对破坏性操作提供清楚确认。
- 新权限必须在 Pull Request 中解释用途。
